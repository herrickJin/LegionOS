# 产研数字军团OS PRD

## 5. 关键场景

> 本章节提取支撑架构设计的核心业务流程与跨域交互，作为技术架构的验证输入。
>
> **核心洞察：任务规范是OS的统一抽象**。项目规范是任务规范的一种类型，普通任务规范可独立执行，子任务规范嵌套在普通任务规范内。场景是任务规范在真实业务中的编排组合。
>
> **场景驱动原则：** 每个场景是用户可感知的端到端价值单元，场景完成后必须可演示、可验证。域是场景的支撑单元，同一个域在不同场景中逐步完善。

### 5.3 核心抽象：任务规范

任务规范是OS的核心抽象，定义"什么人在什么条件下执行什么动作、产出什么、如何验收"。

```
任务规范（Task Spec）
├── 规范标识：ID、名称、版本、类型
├── 规范类型：项目规范 / 普通任务规范 / 子任务规范
│   ├── 项目规范：由MRD触发，包含多个普通任务规范
│   ├── 普通任务规范：可独立执行，由项目规范或人工触发
│   └── 子任务规范：嵌套在普通任务规范内，不可独立触发
├── 输入定义：执行前需要的数据和上下文
├── 输出定义：执行后产出的交付物
├── 验收标准：完成条件和质量门槛
├── 任务流编排：节点序列（人 / 数字员工 / 审核 / 自动化）
│   └── 节点定义：
│       ├── 节点类型：人节点 / 数字员工节点 / 审核节点 / 自动化节点
│       ├── 执行者判定规则：按技能/负载/优先级分配
│       ├── 输入映射：从前序节点或外部获取数据
│       └── 输出映射：产出数据传递给后续节点
├── 判定规则集：每个节点由谁执行的规则
│   └── 规则维度：技能匹配度、当前负载、历史表现、优先级
└── 触发条件：什么事件启动这个任务规范
    ├── MRD已通过（项目规范）
    ├── 人工创建（普通任务规范）
    ├── 前序节点完成（子任务规范）
    └── 定时触发（周期性任务）
```

**关键约束：**
- 项目规范可包含普通任务规范，普通任务规范可包含子任务规范
- 子任务规范不可独立触发，必须由父任务节点触发
- 同一任务规范可多次实例化，每次实例为一个任务执行实例

### 5.4 场景详解

#### SC-01 任务规范的定义与执行

> **场景目标：** 实现"任务规范定义→实例化执行→验收完成"的完整闭环。
>
> **演示的典型任务：**
> - **T1 MRD驱动的项目**（项目驾驶舱）：经营管理者确认MRD后，复合型研发经理在项目驾驶舱中接收MRD、创建项目、查看里程碑和任务进度
> - **T7 代码Review**（Review面板）：AI工程师在Review面板中查看代码Diff、添加行级批注、执行Approve/Reject
>
> **演示标准（第2周末）：**
> - [ ] 能定义任务规范（含6要素）
> - [ ] 能发布任务规范（版本+1，历史保留）
> - [ ] MRD通过后自动触发项目规范实例化
> - [ ] 项目规范实例按节点顺序执行
> - [ ] 普通任务规范可由人工创建并实例化
> - [ ] 任务分配按判定规则执行（技能匹配→人/数字员工）
> - [ ] 节点执行完成后自动流转到下一节点
> - [ ] T1项目驾驶舱可展示MRD、里程碑、任务列表
> - [ ] T7 Review面板可展示Diff、行级批注、Approve/Reject

**业务流程（T1 MRD驱动）：**

```
经营管理者 → 确认MRD通过 → 触发项目规范实例化 → 按节点顺序执行 → 验收完成
```

**关键边界：**
- 任务规范必须关联任务规范，无规范不允许发布
- 项目规范由MRD通过事件触发，普通任务规范由人工创建触发
- 任务分配由产研流程域判定规则驱动，对人和数字员工一视同仁
- 任务流只支持线性执行（MVP），节点按顺序执行
- 节点执行结果必须经人工审核后才能流转到下一节点（审核节点除外）

**域裁剪策略（MVP）：**

| 域 | 裁剪后功能 | 完善度 |
|-----|-----------|--------|
| 产研流程域（wf） | 任务规范CRUD + 线性节点编排 + 判定规则 | 30% |
| 项目实施域（tsk） | 任务实例生命周期 + 节点执行 + 状态流转 | 30% |
| 项目管理域（prj） | 项目信息 + 里程碑（简化） | 20% |
| 经营决策域（opp） | MRD状态 + 触发事件 | 30% |

#### SC-12 数字员工的定义与实例化

> **场景目标：** 实现数字员工"原型定义→灵魂配置→Body部署→实例化→任务执行"的完整链路。
>
> **演示的典型任务：**
> - **T2 Old Java项目梳理**（代码分析面板）：数字员工分析Java代码结构、依赖关系、复杂度
> - **T3 Old Web项目梳理**（代码分析面板）：数字员工分析Web项目页面结构、接口清单
> - **T4 前端代码开发**（轻量IDE面板）：数字员工根据需求生成前端代码
> - **T5 Java代码开发**（轻量IDE面板）：数字员工根据需求生成Java代码
> - **T6 测试用例开发**（测试面板）：数字员工生成测试用例
> - **T8 测试执行**（测试面板）：数字员工执行测试用例
>
> **演示标准（第3周末）：**
> - [ ] 能定义数字员工原型（名称、类型、能力描述）
> - [ ] 能配置灵魂（LLM模型、知识库、工具链）
> - [ ] 能定义Body（运行环境、资源规格）
> - [ ] 能发布数字员工原型
> - [ ] 能将数字员工实例化到项目
> - [ ] 实例化后的数字员工能接收任务节点
> - [ ] T2/T3：数字员工能分析代码，生成分析报告
> - [ ] T5：数字员工能生成Java代码，提交PR

**业务流程：**

```
AI Agent架构师              数字员工域              基础设施域              产研流程域
    │                         │                       │                       │
    │── 创建原型 ────────────→│ 原型:草稿              │                       │
    │   （名称/类型/能力）      │                       │                       │
    │── 配置灵魂 ────────────→│ 灵魂:已配置            │                       │
    │   （LLM/知识库/工具）     │                       │                       │
    │── 定义Body ────────────→│                       │ Body:已定义            │
    │   （环境/资源/脚本）      │                       │                       │
    │── 发布原型 ────────────→│ 原型:已发布            │                       │
    │                         │                       │                       │
    │── 实例化到项目 ─────────→│ 实例:创建中            │                       │
    │                         │── 请求资源 ──────────→│ 分配节点               │
    │                         │←── 资源就绪 ──────────│                       │
    │                         │ 实例:待分配            │                       │
    │                         │                       │                       │
    │                         │←── 任务节点分配 ────────────────────────────────│
    │                         │ 实例:执行中            │                       │
    │                         │── 执行结果 ────────────────────────────────────→│
    │                         │ 实例:闲置              │                       │
```

**关键边界：**
- MVP阶段数字员工执行模式为**远程LLM API调用**，不自研运行时容器
- 实例化时锁定灵魂配置版本，后续灵魂更新不影响已运行实例
- 实例心跳超时自动下线，任务回退到待分配状态
- 资源消耗（token数）记录到节点执行日志

**域裁剪策略（MVP）：**

| 域 | 裁剪后功能 | 完善度 |

#### SC-03 执行评价回流与规范优化
