## 1. 信息架构

### 1.1 导航结构

应用端采用**角色动态导航**——用户登录后根据角色看到不同的侧边栏菜单，共同模块始终可见。

```
应用端
├── 经营管理者视图
│   ├── 经营数据看板（/dashboard/biz）
│   ├── 商机管理（/opportunity）
│   │   ├── 商机列表（/opportunity/list）
│   │   └── 商机详情（/opportunity/:id）
│   ├── 立项审批（/approval）
│   │   ├── 待审批列表（/approval/pending）
│   │   └── 审批详情（/approval/:id）
│   ├── 经营分析（/analytics）
│   └── 资源管理（/resource）
│
├── 复合型研发经理视图
│   ├── 项目管理（/project）
│   │   ├── 项目列表（/project/list）
│   │   └── 项目详情（/project/:id）
│   │       ├── 概览（/project/:id/overview）
│   │       ├── 里程碑（/project/:id/milestones）
│   │       ├── 风险（/project/:id/risks）
│   │       ├── 成员（/project/:id/members）
│   │       └── 关联需求（/project/:id/requirements）
│   ├── 需求管理（/requirement）
│   │   ├── 需求列表（/requirement/list）
│   │   └── 需求详情（/requirement/:id）
│   ├── 任务管理（/task）
│   │   └── 任务看板（/task/board）
│   ├── 进度管控（/progress）
│   │   ├── 项目进度视图（/progress/timeline）
│   │   ├── 风险预警列表（/progress/risks）
│   │   └── 需求变更追踪（/progress/changes）
│   └── 交付收口（/delivery）
│       └── 交付验收（/delivery/:projectId/review）
│
├── AI 工程师视图
│   ├── 个人工作台（/workspace/my）
│   ├── 任务执行（/workspace/my/task/:taskId）
│   ├── 数字员工协同（/ai/collab）
│   │   ├── 执行状态列表（/ai/collab/status）
│   │   └── 执行结果详情（/ai/collab/result/:id）
│   ├── 技术资产（/asset）
│   │   ├── 资产库浏览（/asset/library）
│   │   └── 资产详情（/asset/:id）
│   └── 知识体系（/knowledge）
│
└── 共同模块
    ├── 消息中心（/message）
    ├── 个人设置（/settings）
    └── 组织/项目切换（全局组件，非独立页面）
```

### 1.2 路由规则

- 项目级页面统一携带 `projectId` 作为路径参数或查询参数
- 切换项目上下文时，带 `projectId` 的页面重新请求数据
- 不带 `projectId` 的页面（如个人工作台、消息中心）不受项目切换影响
- 消息跳转：消息条目携带目标路由和参数，点击后导航到对应页面

## 2. 整体布局框架

### 2.1 页面骨架

```
┌─────────────────────────────────────────────────────┐
│ 顶部导航栏                                           │
│ [Logo]  [全局搜索]  [项目切换下拉]  [消息铃铛]  [用户头像▼] │
├──────────┬──────────────────────────────────────────┤
│ 侧边栏    │ 主内容区                                  │
│          │ ┌──────────────────────────────────────┐  │
│ [角色菜单] │ │ 面包屑：首页 > 项目管理 > 项目详情     │  │
│          │ ├──────────────────────────────────────┤  │
│          │ │                                      │  │
│          │ │          页面内容                      │  │
│          │ │                                      │  │
│          │ │                                      │  │
│          │ │                                      │  │
│          │ └──────────────────────────────────────┘  │
│          │                                          │
│          │              （可选：右侧详情抽屉/面板）      │
├──────────┴──────────────────────────────────────────┤
│ [收起/展开侧边栏]                                      │
└─────────────────────────────────────────────────────┘
```

**四个区域**：

**顶部导航栏**（固定高度 56px）
- 左侧：Logo + 产品名称
- 中部：全局搜索（支持搜索项目、任务、需求、人员）
- 右侧：当前项目切换下拉 + 消息通知入口（带未读数 badge）+ 用户头像下拉（个人设置、退出登录）

**侧边栏**（宽度 220px，可收起至 64px）
- 顶部：当前角色标识
- 菜单项按角色视图分组显示
- 支持折叠子菜单
- 底部固定：帮助文档入口

**主内容区**
- 顶部面包屑导航
- 内容区域根据页面类型不同布局不同（列表页、详情页、看板页、看板+详情组合页）

**右侧面板**（按需出现，宽度 400px）
- 任务详情、消息详情等场景下以抽屉形式从右侧滑出
- 不使用时主内容区占满剩余宽度

### 2.2 页面类型模板

应用端的页面分为四类基础布局模板，所有具体页面都在模板上填充内容：

**列表页模板**
- 顶部：页面标题 + 操作按钮（新建等）
- 筛选栏：搜索框 + 条件筛选器 + 排序
- 内容区：数据表格 / 卡片列表（支持分页或无限滚动）
- 批量操作栏（选中条目后底部浮现）

**详情页模板**
- 顶部：返回按钮 + 页面标题 + 操作按钮组
- 内容区：Tab 切换不同维度的信息
- 每个 Tab 内部按模块划分区域

**看板页模板**
- 顶部：页面标题 + 筛选器（按负责人/优先级/项目）
- 内容区：多列看板（列头为状态，卡片为任务）
- 支持拖拽卡片切换列

**看板+详情组合页模板**
- 左侧：看板（占 60% 宽度）
- 右侧：选中卡片的详情面板（占 40% 宽度）
- 点击不同卡片，右侧面板实时刷新

### 3.1 经营数据看板

**页面布局**

```
┌──────────────────────────────────────────┐
│ 经营数据看板          [时间范围：本月 ▼]    │
├──────────┬──────────┬──────────┬──────────┤
│ 营收总额  │ 资源利用率 │ 活跃项目数 │ 风险项目数 │
│ ¥xxx万   │ 78%      │ 12       │ 2        │
│ ↑12%    │ ↑5%     │ ↑1      │ ↓1      │
├──────────┴──────────┴──────────┴──────────┤
│ ┌─────────────────┐ ┌─────────────────┐  │
│ │ 营收趋势（折线图） │ │ 项目健康度分布    │  │
│ │                 │ │ （饼图）         │  │
│ │    [图表区域]     │ │    [图表区域]     │  │
│ └─────────────────┘ └─────────────────┘  │
│ ┌─────────────────┐ ┌─────────────────┐  │
│ │ 资源负载分析     │ │ 项目投入产出      │  │
│ │ （柱状图）       │ │ （散点图）       │  │
│ │    [图表区域]     │ │    [图表区域]     │  │
│ └─────────────────┘ └─────────────────┘  │
└──────────────────────────────────────────┘
```

- 顶部指标卡片行：4 个核心 KPI，显示当前值和环比变化
- 下方图表区：2x2 网格，每个图表可点击全屏查看

**交互流程**

1. 用户进入看板，默认加载本月数据
2. 切换时间范围（本周/本月/本季度/本年度），所有指标和图表刷新
3. 点击指标卡片，下钻到对应维度的分析页（如点击"风险项目数"跳转经营分析页的风险维度）
4. 点击图表中的某个数据点，弹出 tooltip 显示明细

**数据流向**

- 前端加载 → `GET /api/dashboard/biz-stats?period=month` → 返回指标数据和图表数据
- 时间范围切换 → 重新请求同接口，携带不同 `period` 参数
- 无实时推送需求（看板数据定时轮询即可，30秒间隔）


- Tab 切换：基本信息、评估信息（评分维度+打分）、转化记录（状态变更历史）、操作日志
- 状态流转按钮在详情页右上角，根据当前状态显示可用的下一步操作

**交互流程**

1. 从商机列表进入详情页
2. 编辑商机信息（点击"编辑"进入编辑模式，或直接在详情页内联编辑）
3. 状态流转：录入→评估→转化→立项/关闭，每步需要填写对应信息
4. 转化为立项：选择"转化为立项"，系统创建项目并跳转到项目管理页

**数据流向**

- 列表加载 → `GET /api/opportunity/list?status=&page=&page_size=`
- 详情加载 → `GET /api/opportunity/:id`
- 状态变更 → `POST /api/opportunity/:id/status` → 服务端处理 → `SSE` 推送状态变更通知给相关人员
- 商机转立项 → `POST /api/opportunity/:id/convert-to-project` → 返回新项目 ID → 前端路由跳转到项目详情

## 7. 数据流向总览

### 7.1 交互模式分类

应用端与 Server 层的交互按模式分为四类：

**模式 A：标准 REST 请求-响应**

页面加载、表单提交、CRUD 操作。绝大多数页面功能属于此类。

前端 → `GET/POST/PUT/DELETE /api/*` → Server → 返回 JSON

**模式 B：SSE 实时推送**

需要服务端主动推送信息的场景：

| 场景 | SSE 通道 | 触发条件 |
|------|----------|---------|
| 消息通知 | `/api/message/stream` | 有新系统通知、预警提醒、@消息 |
| 任务状态变更 | 由消息中心统一推送 | 任务被分配、状态变更、被验收 |
| 数字员工流式输出 | `/api/ai/collab/stream/:id` | 数字员工正在执行生成任务 |
| 风险预警 | 由消息中心统一推送 | 系统检测到延期/偏差信号 |
| 审批结果 | 由消息中心统一推送 | 审批通过/驳回 |

**模式 C：文件上传**

产出物提交、资产贡献、头像上传等场景。

前端 → `POST /api/*` (multipart/form-data) → Server → 文件存储服务 → 返回文件 URL

**模式 D：文件下载/导出**

报告导出、资产下载等场景。

前端 → `GET /api/*/export` → Server → 返回文件流（PDF/Excel）

### 7.2 实时推送架构

```
用户浏览器
    │
    ├── SSE 连接 1：/api/message/stream（所有推送消息统一通道）
    │       接收：任务变更、审批结果、风险预警、@消息、系统通知
    │
    ├── SSE 连接 2：/api/ai/collab/stream/:executionId（数字员工流式输出，按需建立）
    │       接收：数字员工生成内容（逐 chunk 推送）
    │
    └── 标准HTTP 请求（页面加载、表单提交等按需发起）
```

设计要点：

- 推送消息统一走一个 SSE 通道，前端根据消息类型分发到不同处理器
- 数字员工流式输出是独立 SSE 通道，只在查看执行结果时建立连接，执行完成或离开页面时断开
- 不使用 WebSocket（WebSocket 通道留给数字员工节点之间的内部通信）
