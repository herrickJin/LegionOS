# 产研数字军团OS — 术语定义表

---

产研数字军团OS — 构建在技术资产、知识体系、项目库、业务系统之上，通过 AI Agent 调度资源、适配流程，为四大角色提供从商机洞察到交付运营的全流程产研能力支持的操作系统级平台

数字员工 — 在 OS 中承担某一角色、具备特定思维模式和技能的 AI Agent，与人类共享上下文执行标准化任务，人始终对最终交付结果承担责任

产研流程规范 — OS 的核心抽象，定义什么角色在什么条件下执行什么动作、产出什么、如何验收，覆盖从商机洞察到交付评估的全链路，对人和数字员工一视同仁

任务流规范 — 产研流程规范的执行层抽象，包含元数据、输入项与标准、产出项与验收标准、子任务清单与任务流、执行岗责任要求、执行工作台六要素

---

经营管理者 — 对部门经营结果负责的全链路决策者

复合型研发经理 — 连接经营目标与技术执行的枢纽角色，负责项目交付与团队效能

AI 工程师 — 利用 AI 工具与数字员工协同完成具体研发任务的执行角色

AI Agent 架构师 — 负责 OS 能力迭代、数字员工优化、技术资产运营的架构角色

---

数字员工节点 — Agent Communication Proxy（ACP），跑在工程师终端上的守护进程，凌驾于所有本地 Agent 之上，Rust 实现。职责：管理 Agent 子进程生命周期、Server 与 Agent 之间的协议翻译与消息路由、流式输出实时转发。不管调度、执行、缓存、持久化

Agent — 在终端上运行的 AI 工具，如 Hermes、Claude Code 等。由 ACP 以子进程方式启动和管理（CLI 类型），或以 HTTP 长连接方式交互（HTTP API 类型）

---

iam-svc — Identity and Access Management service，类 EHR 系统的用户体系服务。管理人类用户和数字员工实例的生命周期、组织层级（部门归属 + 项目成员矩阵）、RBAC 功能权限（菜单/按钮级）。数字员工也是用户，功能权限由本域管理，业务能力由数字员工域配置决定。数据权限（行级）由各业务域自行管理。

pdm-svc — MVP 阶段的核心聚合服务，聚合经营决策、项目管理、项目实施、产研流程四个域

asset-svc — MVP 阶段的资产聚合服务，聚合技术资产、知识经验、自我进化、价值评估四个域

agent-svc — MVP 阶段的数字员工服务，负责数字员工原型管理、灵魂配置、实例管理

gateway-svc — 技术网关服务（Spring Cloud Gateway），外部流量统一入口，负责路由、JWT 验签、限流、熔断、WebSocket 升级，依赖 Redis（限流计数、JWT 黑名单）

portal-svc — BFF 层（Backend for Frontend），成长期引入。为前端提供个人工作台、仪表盘等跨域聚合数据，依赖 Redis 缓存。MVP 阶段不引入，前端通过 gateway-svc 直接调业务服务

Gateway — Spring Cloud Gateway，OS 服务端统一入口（gateway-svc），负责路由、JWT 验签、限流、熔断、WebSocket 升级

---

C4 模型 — 软件架构图的四层表达方法：系统上下文、容器图、组件图、代码图

Context Map — DDD 上下文映射图，描述多个限界上下文之间的集成关系（Customer-Supplier、Conformist、ACL）

BFF — Backend for Frontend，为特定前端定制的数据聚合服务

RBAC — Role-Based Access Control，基于角色的访问控制

Saga — 分布式事务模式，用于跨服务的长事务协调，MVP 阶段使用本地事务，成长期引入

RAG — Retrieval-Augmented Generation，检索增强生成，使数字员工能基于组织知识库回答问题

Event Sourcing — 事件溯源，将系统状态变化记录为不可变事件序列，可审计、可回放、可学习

CQRS — Command Query Responsibility Segregation，命令查询职责分离

WebSocket — 全双工通信协议，走标准 HTTP 端口，作为数字员工节点与 Server 之间的实时通信通道，用于任务推送和状态上报

---

MVP — 最小可行产品，OS 的 MVP 阶段为 4 周

MRD — 市场需求文档，审批通过后触发项目规范的实例化

PRD — 产品需求文档，需求确认阶段的核心交付物

ADR — Architecture Decision Record，架构决策记录，记录关键决策的上下文、选项、理由和后果

SC — 关键场景，OS 中用户可感知的端到端价值单元，当前定义 12 个

LLM API — 大语言模型接口，MVP 阶段数字员工通过远程 LLM API 执行，不自研运行时

Vector DB — 向量数据库，用于存储资产和知识的向量表示，支持 RAG 检索，后期引入

TBC — To Be Confirmed，待确认
