# 核心流程规范

| 编号  | 规范名称        | 场景说明                                                                                                    |
| --- | ----------- | ------------------------------------------------------------------------------------------------------- |
| T1  | 遗留前端项目分析    | 接手陌生前端仓库，1自动产出 组件树/路由/状态管理全景 + 技术债清单。交互台展示扫描进度、组件树可视化、可编辑分析报告。                                          |
| T2  | 遗留后端项目分析    | 接手陌生后端仓库，自动产出 模块依赖图/循环依赖/DB Schema 演化历史 + 接手建议。交互台展示依赖图可视化、问题清单、可编辑接手报告。                                |
| T3  | 详细设计驱动的前端开发 | 前端开发依据详细设计文档直接实现，减少返工。交互台左侧展示详细设计文档（组件 spec/交互流程/状态设计），右侧为代码编辑区 + UI 预览。                                |
| T4  | 详细设计驱动的后端开发 | 后端开发依据详细设计文档直接实现 API 和数据层，减少联调扯皮。交互台左侧展示详细设计文档（API 定义/数据模型/业务逻辑流），右侧为代码编辑区 + 接口调试面板。                    |
| T5  | 测试用例编写      | 需求或详细设计出来后，自动识别测试场景并生成用例模板。交互台展示需求/设计文档原文、自动生成的测试场景列表、可编辑用例表格。                                          |
| T6  | 代码 Review   | PR 提交后，Reviewer 在统一界面完成 Diff 查看、行级批注、Approve/Reject。交互台展示 Diff 查看器、行级批注输入区、Approve/Reject 操作区、数字员工预审意见。 |
| T7  | 测试执行        | 测试用例编写完成后，自动触发回归测试并对比历史结果。交互台展示用例执行列表、实时执行日志、通过/失败/跳过状态、执行报告导出。                                         |
| T8  | 需求层设计       | 结构化需求（T15 输出）输入后，自动产出业务流程图、功能需求清单、可点击交互原型。交互台左侧为需求原文，右侧为流程图/清单/原型的三Tab 工作台。                             |
| T9  | 架构层设计       | PRD 输入后，自动生成技术选型候选方案对比表、系统架构图草稿。交互台展示候选方案对比表、架构图编辑器、部署架构草稿。                                             |
| T10 | 概要设计        | 功能清单输入后，自动生成模块划分草案、接口定义草稿、时序设计。交互台展示模块划分图、接口列表编辑器、时序图编辑器。                                               |
| T11 | 详细设计        | 概要设计定完后，自动生成类骨架代码、数据库 DDL、状态机代码框架。交互台展示类图编辑器、DDL 预览、状态机可视化、代码框架导出。                                      |
| T12 | 专项设计        | 对照安全/性能/可观测性/国际化 checklist 自动扫描设计文档盲区。交互台展示四项专项的 checklist 扫描结果、改进建议列表、设计文档关联视图。                        |
| T13 | 设计验证与交付     | 多份设计文档之间的一致性自动检查并标记偏差。交互台展示设计文档间的一致性检查报告、偏差标记列表、交付物导出区。                                                 |
| T14 | 日志文件分析      | 线上出问题后，上传日志自动聚类异常、关联时间线、定位根因。交互台展示日志文件上传区、异常聚类时间线图、根因定位报告、修复建议。                                         |
| T15 | 需求结构化与准入    | 模糊需求（聊天记录/语音/截图）输入后，输出结构化需求文档 + 准入 checklist。交互台左侧为原始需求 + 数字员工对话，右侧为结构化需求可编辑文档 + 准入评估。                  |
| T16 | PRD 任务分解    | PRD 出来后，自动生成功能清单 → 拆解为可执行任务卡片 → 生成里程碑。交互台展示功能清单、任务卡片看板、里程碑时间线，支持拖拽调整。                                   |
| T17 | 详细设计驱动的终端开发 | CLI/脚本工具开发依据详细设计文档直接实现命令结构和参数处理。交互台左侧展示详细设计文档（命令结构/输入输出格式），右侧为代码编辑区 + 终端输出预览。                           |
| T18 | 技术方案评审      | 设计文档完成后，数字员工对照评审 checklist 扫描并生成评审意见草稿。交互台展示设计文档、评审 checklist 扫描结果、评审意见草稿、ADR 编辑器。                      |
| T19 | 发布与回滚       | 发布前自动执行 checklist，发布后自动盯指标 15 分钟，异常自动建议回滚。交互台展示发布 checklist、实时指标监控面板、回滚操作区、发布记录时间线。                     |
| T20 | 缺陷管理        | Bug 提交后自动分级、建议分配人、跟踪修复→回归→关闭闭环。交互台展示缺陷列表（按 P0-P3 分级）、分配看板、趋势分析图表。                                       |
| T21 | 接口契约管理      | 接口变更时自动检测破坏性变更，列出所有消费者并发送通知。交互台展示接口契约列表、变更检测报告、消费者通知状态跟踪、版本演化时间线。                                       |
| T22 | 故障应急响应      | 线上告警触发后，自动创建故障频道、按 on-call 排班分配角色、拼时间线。交互台展示故障时间线、角色分工面板、操作日志、复盘草稿编辑器。                                  |
| T23 | 可行性评估       | 需求结构化完成后，从技术可行/工期量级/资源缺口三个维度自动出评估报告。交互台展示历史项目数据参考、团队负载视图、评估报告草稿。                                        |
| T24 | 交付承诺定稿      | 可行性评估通过后，将范围/时间/资源/变更规则形成双方认可的交付承诺清单。交互台展示承诺清单编辑器、历史偏差数据对比、双方确认状态跟踪。                                    |
| T25 | 需求变更管理      | 执行中变更请求进来，自动对比原始承诺清单并标记影响范围。交互台展示变更请求列表、影响范围可视化、决策链时间线、原始承诺对比视图。                                        |
| T26 | MRD 自动生成    | 商机确认后，自动聚合商机情报 + 可行性评估 + 竞品分析，生成结构化 MRD。交互台展示 MRD 模板各章节编辑器、历史 MRD 参考、导出 Word/PPT 操作区。                   |
| T27 | 技术资产识别与入库   | 项目完成后自动扫描可复用组件/工具/模板，评估入库价值。交互台展示资产候选清单、复用价值评估、入库审批流、新项目匹配推荐列表。                                         |
| T28 | 知识体系沉淀      | Bug 复盘/架构决策/踩坑记录自动从项目过程中提取并沉淀到知识库。交互台展示知识条目编辑器、过期检测列表、新任务知识推荐面板、FAQ 管理区。                                |
| T29 | 质量审核        | 关键流程/高影响变更/核心链路提交后，数字员工对照质量标准逐项扫描。交互台展示质量审核 checklist 扫描结果、偏差标记列表、审核结论编辑器、放行/驳回操作区。                     |
